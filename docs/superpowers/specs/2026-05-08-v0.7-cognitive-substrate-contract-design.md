## 1. Hard problem

v0.6.2's evaluator checks the spec's *technical contract*: paths, packages, routes, DB tables, negative paths, integrity hashes. It is silent on the spec's *cognitive-substrate contract* ‚Äî the eight axes that determine whether a spec is complete enough to be executed safely by an LLM implementer with a human in the loop:

1. **Receiver fingerprint** ‚Äî which engine the spec is calibrated to.
2. **Trust boundaries** ‚Äî untrusted-input flow, secrets, contextual binding.
3. **UX/observability contract** ‚Äî what the operator sees on success/failure, where logs land, halt remediation.
4. **Possibility-walk** ‚Äî assumptions enumerated and killed.
5. **Metis-boundary** ‚Äî which steps depend on situated judgment that cannot be specified.
6. **Adversarial-use map** ‚Äî exploit pathways through untrusted data.
7. **Net cognitive ROI** ‚Äî yield-curve slope vs. scaffolding cost.
8. **Provenance chain** ‚Äî derivation history; bound to envelope.

Three vault concepts make these axes load-bearing rather than nice-to-have:

- *"Mutual distrust between cognitive nodes."* Specs cannot treat upstream context as authoritative without verification. Today, ¬ß8.1 verifies file system contracts but not cognitive ones.
- *"Possibility-space exploration must be made explicit."* Humans prune via somatic priors; LLMs do not. Spectre has `negative-paths:` per step but no spec-wide possibility audit.
- *"Lossy summary as attack surface."* Handoffs hide critical safety checks in compressed-away portions. v0.6's envelope binds bytes but not semantics.

Without ¬ß8.2, every spec passes Tier 1/2/3 with the cognitive contract entirely unverified. The v0.6.1 retest of activity-ingestion-daemon already exposed this ‚Äî Tier 3 silently degraded for auth reasons (issue #37) AND nothing in the spec called out untrusted-input handling on the network ingestion paths.

## 2. First principles

- **Determinism beats probabilism for safety-critical checks.** Trust-boundary taint flow goes in Tier 1 (deterministic), not Tier 3 (probabilistic). Tier 3 advisory only.
- **The receiver is fixed per /vision invocation.** A spec is calibrated to one receiver; receiver fingerprint is not a runtime variable.
- **Ceremony has to earn its place.** Every new field must catch a real failure mode; if forcing every step to declare it produces fill-in-the-blank bookshow, the field is wrong.
- **No manual invocation.** The author should not have to remember to add ¬ß8.2; /vision must fire the wizard automatically.
- **Backward compat.** Pre-v0.7 locked specs continue to work; their /implement halts produce a single warn finding, not a block.
- **Defense in depth.** Tier 1 deterministic + Tier 3 advisory + Tier 0 envelope check. No single layer is load-bearing alone.

## 3. Algorithm audit

- **Delete:** nothing. v0.7 is purely additive.
- **Simplify:** receiver fingerprint goes through a wizard, not a free-form field ‚Äî author writes nothing for it. Same for trust profile.
- **Accelerate:** Tier 1 substrate_ast runs in <100ms (taint flow is O(steps¬≤) in the worst case but steps ‚â§30, matching the existing walker `max_rounds` ceiling in `~/.spectre/walker.toml`). Tier 3 adds a single ~120-token rubric; no per-step API fanout.

## 4. Speed-of-light limit

The fastest version of v0.7: wizard fires once per /vision (4 questions, ‚â§30s author time), substrate_ast runs in ‚â§100ms, Tier 3 adversarial-pathway adds ‚â§2s to the existing call (no extra round-trip ‚Äî rubric is appended to the contradiction-tuple prompt, single batched call). Total /vision overhead: <40 seconds for the wizard, <2 seconds for evaluation; envelope write adds ‚â§10ms.

## 5. Physics guardrails

- Wizard answers are cached at `~/.spectre/substrate-cache/<author-spec-hash>.json` keyed by the **author-spec hash** (SHA-256 over the draft body minus the auto-injected `### 8.2` block ‚Äî per Copilot review point 3 ‚Äî so wizard injection doesn't invalidate its own cache); re-running /vision on unchanged author body = no re-prompt.
- ¬ß8.2 bytes are SHA-256-locked into `<slug>.envelope.json` as `substrate_sha256`.
- Wizard schema version is recorded; mismatch forces re-prompt and emits `cognitive-substrate-stale` info finding.
- No new daemons. No new network calls. Stdlib only.
- Per-step trust annotations are evaluator-time only; the executor does not enforce them at runtime (deferred to v0.8 if needed).

### 6.3 ¬ß8.2 schema (canonical)

Severities split: **block** for fields with deterministic semantics; **warn** for subjective fields where machine enforcement would optimise for form-filling rather than safety (per Copilot review, point 5).

```markdown
### 8.2 Cognitive-substrate contract

# block-severity (machine-enforced ‚Äî must be present + non-empty)
- receiver-fingerprint: <auto-filled by wizard>
- trust-profile: <auto-filled, multi-set>
- contextual-binding: <one-line description of what this spec is FOR>
- provenance: <{ kind: derived-from, parent-envelope-sha256: <hex64>, parent-slug: <slug> } | { kind: none }>
- ux-contract:
    on-success: <one-line operator-visible message>
    on-failure: <one-line operator-visible message + remediation hint>
    log-target: <path or stream>

# warn-severity (informational; block only when assumptions-killed empty AND steps>3)
- assumptions-killed: <list of considered-and-ruled-out alternatives>
- requires-situated-judgment: <list of step IDs; cap = max(1, floor(0.3 √ó n_steps))>
- roi-budget: <yield-curve slope target / scaffolding cost ceiling>
```

**Provenance binding (Copilot point 6).** `derived-from:` carries both the parent slug AND the parent envelope SHA-256 hash. Tier 1 verifies the parent envelope exists and the hash matches; mismatch ‚Üí block `provenance-broken`. Slug-only declarations from earlier drafts ‚Üí warn `provenance-weak-binding`.

### 6.4 Per-step trust annotations (taint flow)

**Annotations are MANDATORY when the wizard's `trust-profile:` includes `untrusted-input` or `handles-secrets`.** Copilot review point 1: silent-false-negative-by-default is wrong for a safety feature. The wizard answer is the gate ‚Äî if the spec author declares untrusted input, every step that produces or consumes a file/db-table/route artifact MUST declare `untrusted-input:` (yes/no). Missing annotation on such a spec ‚Üí block `trust-annotation-required`.

**Source/sanitized-output model (Copilot point 2).** `sanitizes:` clears taint on the OUTPUT the step `produces:`, NOT the input. Source artifacts stay tainted forever; downstream consumers must either sanitize before reaching a sink, or read from a sanitized output instead.

**Sink categories (Copilot point 2).** Sinks are not just `mutates:` paths. Tier 1 recognises four sink categories, each as a separate detector function in `bin/substrate_ast.py`:

1. **Filesystem-write sinks** ‚Äî `mutates:` paths from ¬ß8.1.
2. **Shell-eval sinks** ‚Äî action tokens that re-evaluate tainted content: `bash -c "$(cat ‚Ä¶)"`, `sh -c "$(‚Ä¶)"`, eval-like forms.
3. **SQL/template sinks** ‚Äî action tokens matching SQL DML keywords (`INSERT`, `UPDATE`, `REPLACE`) or template-render verbs (`jinja2`, `template_render`, formatted-string interpolation).
4. **Network-egress sinks** ‚Äî `curl|wget|http(ie)? POST|PUT` with body content sourced from a tainted produces.

Each detector emits `untrusted-flow-unguarded` block with the sink kind in the finding message.

```yaml
- step: 3
  why: "fetch external JSON"
  action: "curl -fsSL https://example.com/data > /tmp/x.json"
  verification: "test -f /tmp/x.json"
  produces: ["file:/tmp/x.json"]
  untrusted-input: "yes"
  halt-hint: "fetch failed: check network or signed-URL expiry"

- step: 4
  why: "extract + sanitize title"
  action: "jq '.title' /tmp/x.json | sanitize-html > /tmp/y.txt"
  verification: "test -f /tmp/y.txt"
  produces: ["file:/tmp/y.txt"]
  sanitizes: ["file:/tmp/y.txt"]
  requires: ["file:/tmp/x.json"]
```

**Taint flow rule (revised).** Taint propagates from any `untrusted-input: yes` step's produces to every downstream step's produces UNLESS that step's `sanitizes:` listing names the OUTPUT artifact. Reaching any of the four sink categories above with tainted artifacts ‚Üí block `untrusted-flow-unguarded`.

## 10. Verification

1. **Unit tests:** `pytest tests/ -v` baseline 1280 passes; v0.7 adds ~38 new test functions.
2. **End-to-end wizard run:** in a fresh test project, run /vision with all 4 wizard answers; confirm ¬ß8.2 auto-injected; sidecar has `substrate_resolution`; envelope has `substrate_sha256`.
3. **Tampering test:** lock spec, modify ¬ß8.2 bytes; run /implement ‚Üí Tier 0 block `envelope-tampered: substrate-bytes-changed`.
4. **Taint flow test:** lock spec with step-3 `untrusted-input: yes` + step-5 mutates without sanitization ‚Üí Tier 1 block `untrusted-flow-unguarded`. Add `sanitizes:` step in between ‚Üí lock succeeds.
5. **Backward compat:** locked v0.6.2 spec (envelope without `substrate_sha256`) passes /implement Tier 0 with one warn finding `envelope-missing-substrate`, no halt.

## Peer Review Notes (Copilot/GPT-5.4)

Reviewed 2026-05-08. Six issues raised; five acted on, one partially declined.

- **Acted (1 ‚Äî silent-false-negative-by-default).** ¬ß6.4 now mandates per-step `untrusted-input:` annotations whenever the wizard's `trust-profile:` includes `untrusted-input` or `handles-secrets`. Missing annotation on such a spec is block, not warn.
- **Acted (2 ‚Äî taint model coherence).** ¬ß6.4 rewritten: `sanitizes:` clears taint on the OUTPUT, not the source. Source artifacts stay tainted. Sink model expanded from a single category (`mutates:`) to four detector functions: filesystem-write, shell-eval, SQL/template, network-egress. Each with its own emitter.
- **Acted (3 ‚Äî cache hash stability).** Wizard cache is keyed by the **author-spec hash** (SHA-256 over draft body minus the auto-injected ¬ß8.2 block), not the full spec hash. Wizard injection no longer invalidates its own cache.
- **Acted (4 ‚Äî state location).** Cache moves from `state/.substrate.json` (committed by default) to `~/.spectre/substrate-cache/<author-spec-hash>.json`. Matches the existing `~/.spectre/` convention for `reviewer.toml`, `walker.toml`.
- **Partially acted (5 ‚Äî block on subjective fields).** Demoted `roi-budget` and the `requires-situated-judgment` cap to warn (would optimise for form-filling). Kept `assumptions-killed` empty as block on specs with >3 steps because the essay's possibility-walk discipline is the load-bearing point of having the field at all; without that gate it becomes optional advice.
- **Acted (6 ‚Äî provenance binding).** `derived-from:` now carries both parent-slug AND parent-envelope-sha256. Tier 1 verifies the hash matches the live parent envelope; mismatch ‚Üí block. Slug-only declarations ‚Üí warn `provenance-weak-binding`.
