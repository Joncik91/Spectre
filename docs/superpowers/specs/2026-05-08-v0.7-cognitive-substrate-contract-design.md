# Spectre v0.7 — Cognitive-Substrate Contract (§8.2) Design

**Status:** design (pre-plan).
**Date:** 2026-05-08.
**Author:** dogfooded /vision simulation; vault-informed.
**Predecessors:** v0.6.2 (issues #36, #37 closed). v0.6.0 envelope + walker yield + negative-paths + Tier 3 faithfulness. v0.5.2 step contracts + executor-owned venv.
**Vault sources:** `concepts/context-as-cognitive-substrate.md`, `concepts/heidegger-predicted-gen-ai-comparison.md`.

## 1. Hard problem

v0.6.2's evaluator checks the spec's *technical contract*: paths, packages, routes, DB tables, negative paths, integrity hashes. It is silent on the spec's *cognitive-substrate contract* — the eight axes that determine whether a spec is complete enough to be executed safely by an LLM implementer with a human in the loop:

1. **Receiver fingerprint** — which engine the spec is calibrated to.
2. **Trust boundaries** — untrusted-input flow, secrets, contextual binding.
3. **UX/observability contract** — what the operator sees on success/failure, where logs land, halt remediation.
4. **Possibility-walk** — assumptions enumerated and killed.
5. **Metis-boundary** — which steps depend on situated judgment that cannot be specified.
6. **Adversarial-use map** — exploit pathways through untrusted data.
7. **Net cognitive ROI** — yield-curve slope vs. scaffolding cost.
8. **Provenance chain** — derivation history; bound to envelope.

Three vault concepts make these axes load-bearing rather than nice-to-have:

- *"Mutual distrust between cognitive nodes."* Specs cannot treat upstream context as authoritative without verification. Today, §8.1 verifies file system contracts but not cognitive ones.
- *"Possibility-space exploration must be made explicit."* Humans prune via somatic priors; LLMs do not. Spectre has `negative-paths:` per step but no spec-wide possibility audit.
- *"Lossy summary as attack surface."* Handoffs hide critical safety checks in compressed-away portions. v0.6's envelope binds bytes but not semantics.

Without §8.2, every spec passes Tier 1/2/3 with the cognitive contract entirely unverified. The v0.6.1 retest of activity-ingestion-daemon already exposed this — Tier 3 silently degraded for auth reasons (issue #37) AND nothing in the spec called out untrusted-input handling on the network ingestion paths.

## 2. First principles

- **Determinism beats probabilism for safety-critical checks.** Trust-boundary taint flow goes in Tier 1 (deterministic), not Tier 3 (probabilistic). Tier 3 advisory only.
- **The receiver is fixed per /vision invocation.** A spec is calibrated to one receiver; receiver fingerprint is not a runtime variable.
- **Ceremony has to earn its place.** Every new field must catch a real failure mode; if forcing every step to declare it produces fill-in-the-blank bookshow, the field is wrong.
- **No manual invocation.** The author should not have to remember to add §8.2; /vision must fire the wizard automatically.
- **Backward compat.** Pre-v0.7 locked specs continue to work; their /implement halts produce a single warn finding, not a block.
- **Defense in depth.** Tier 1 deterministic + Tier 3 advisory + Tier 0 envelope check. No single layer is load-bearing alone.

## 3. Algorithm audit

- **Delete:** nothing. v0.7 is purely additive.
- **Simplify:** receiver fingerprint goes through a wizard, not a free-form field — author writes nothing for it. Same for trust profile.
- **Accelerate:** Tier 1 substrate_ast runs in <100ms (taint flow is O(steps²) in the worst case but steps ≤30, matching the existing walker `max_rounds` ceiling in `~/.spectre/walker.toml`). Tier 3 adds a single ~120-token rubric; no per-step API fanout.

## 4. Speed-of-light limit

The fastest version of v0.7: wizard fires once per /vision (4 questions, ≤30s author time), substrate_ast runs in ≤100ms, Tier 3 adversarial-pathway adds ≤2s to the existing call (no extra round-trip — rubric is appended to the contradiction-tuple prompt, single batched call). Total /vision overhead: <40 seconds for the wizard, <2 seconds for evaluation; envelope write adds ≤10ms.

## 5. Physics guardrails

- Wizard answers are cached at `state/.substrate.json` keyed by the spec hash; re-running /vision on unchanged spec = no re-prompt.
- §8.2 bytes are SHA-256-locked into `<slug>.envelope.json` as `substrate_sha256`.
- Wizard schema version is recorded; mismatch forces re-prompt and emits `cognitive-substrate-stale` info finding.
- No new daemons. No new network calls. Stdlib only.
- Per-step trust annotations are evaluator-time only; the executor does not enforce them at runtime (deferred to v0.8 if needed).

### 6.3 §8.2 schema (canonical)

```markdown
### 8.2 Cognitive-substrate contract (machine-enforced — `block` severity)

- `receiver-fingerprint:` <auto-filled by wizard: claude-code+human / claude-code-autonomous / non-claude-ai / human-only>
- `trust-profile:` <auto-filled, multi-set: untrusted-input | handles-secrets | touches-network | executes-generated-code | none>
- `contextual-binding:` <one-line description of what this spec is FOR; refusing replay as something else>
- `provenance:` <derived-from: <slug> | none>
- `assumptions-killed:` <list of considered-and-ruled-out alternatives, one per line>
- `requires-situated-judgment:` <list of step IDs where the implementer must halt + ask, not guess. Cap = max(1, floor(0.3 × n_steps)). For 3 steps, max 1 claim; for 10 steps, max 3.>
- `ux-contract:`
    on-success: <one-line operator-visible message>
    on-failure: <one-line operator-visible message + remediation hint>
    log-target: <path or stream>
- `roi-budget:` <yield-curve slope target / scaffolding cost ceiling, e.g. "≥0.3 yield per 1k tokens">
```

### 6.4 Per-step trust annotations (optional, used by taint flow)

```yaml
- step: 3
  why: "..."
  action: "curl -fsSL https://example.com/data > /tmp/x.json"
  verification: "test -f /tmp/x.json"
  produces: ["file:/tmp/x.json"]
  untrusted-input: "yes"        # NEW
  halt-hint: "fetch failed: check network or signed-URL expiry"  # NEW

- step: 4
  why: "..."
  action: "jq '.title' /tmp/x.json | sanitize-html > /tmp/y.txt"
  verification: "test -f /tmp/y.txt"
  produces: ["file:/tmp/y.txt"]
  sanitizes: ["file:/tmp/x.json"]   # NEW — clears taint from listed produces
  requires: ["file:/tmp/x.json"]
```

Taint flow rule: if step N declares `untrusted-input: yes` and step N+M writes to a `mutates:` path, AND no step between N and N+M declares `sanitizes:` covering N's produces, emit `untrusted-flow-unguarded` block.

## 10. Verification

1. **Unit tests:** `pytest tests/ -v` baseline 1280 passes; v0.7 adds ~38 new test functions.
2. **End-to-end wizard run:** in a fresh test project, run /vision with all 4 wizard answers; confirm §8.2 auto-injected; sidecar has `substrate_resolution`; envelope has `substrate_sha256`.
3. **Tampering test:** lock spec, modify §8.2 bytes; run /implement → Tier 0 block `envelope-tampered: substrate-bytes-changed`.
4. **Taint flow test:** lock spec with step-3 `untrusted-input: yes` + step-5 mutates without sanitization → Tier 1 block `untrusted-flow-unguarded`. Add `sanitizes:` step in between → lock succeeds.
5. **Backward compat:** locked v0.6.2 spec (envelope without `substrate_sha256`) passes /implement Tier 0 with one warn finding `envelope-missing-substrate`, no halt.

## 11. Out of scope (deferred)

- **Runtime taint enforcement** by the executor (v0.8 candidate; v0.7 is evaluator-time only).
- **Free-form receiver fingerprint** with Tier 3 reading it for tailored review (v0.7 fixes receiver to one of four wizard tiers).
- **Multi-spec ROI ledger** (`~/.spectre/roi.jsonl` cross-spec analytics; v0.7 records per-spec ROI in the sidecar but doesn't aggregate).
- **Provenance chain visualization** (CLI to render derived-from DAG; defer until 5+ derived specs exist).
- **Wizard answer migration** when a spec is forked (manual re-run of /vision is the v0.7 path).
